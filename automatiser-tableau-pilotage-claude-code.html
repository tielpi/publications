01Pré-requis

Cette page est le prolongement technique du mode d'emploi. Plus avancée, mais toujours pas à pas. Si la consigne permanente du mode d'emploi vous suffit, vous pouvez l'ignorer sans rien perdre d'essentiel.

  • Vous utilisez Claude Code (le mode d'emploi de base, lui, marche avec n'importe quel assistant).
  • Vous avez déjà le tableau de bord de base : le dossier pilotage/ avec index.html et donnees.js.
  • Votre travail est rangé dans des dossiers thématiques (chez nous bibliotheque/, notes/, recherches/, projets/). Vous adapterez ces noms aux vôtres.

02Le principe : séparer trois rôles

Tenir un tableau à jour mêle trois choses différentes, qu'on gagne à distinguer.

Le déclencheur

Ce qui fait qu'on pense à mettre à jour. Sans lui, on oublie. → le hook.

La méthode

Comment mettre à jour correctement : quoi scanner, quoi vérifier. → la skill.

La source de vérité

Les données elles-mêmes. → donnees.js, inchangé.

Le mode d'emploi de base fusionne les deux premiers dans une consigne écrite. Les outils de cette page les séparent proprement, chacun à sa place, chacun réutilisable. C'est la même idée que la progressive disclosure : ne montrer une information qu'au moment où elle sert, plutôt que d'encombrer en permanence les instructions de l'assistant.

03Le hook — le déclencheur automatique

Un hook est un petit script que Claude Code exécute tout seul, à un moment précis, sans que l'assistant ait à y penser.

Le moment qui nous intéresse s'appelle l'événement Stop : la fin de chaque échange. À cet instant, notre hook vérifie si le tableau a pris du retard, et affiche un rappel discret si c'est le cas. C'est un réveille-matin, pas un exécutant : il sonne, il ne range pas la chambre.

Le code, commenté

En clair : il repère la source de vérité donnees.js et la lit ; il parcourt chaque sous-dossier de bibliotheque/, notes/, recherches/ contenant au moins une note ; pour chacun, il vérifie si son nom apparaît dans donnees.js. S'il en manque un, il l'ajoute à une liste et affiche un rappel. Si tout est référencé, il se tait.

Ses deux angles morts, à connaître

  • Il ne voit que l'omission, pas la péremption. Il détecte un dossier absent, mais il est aveugle à un chantier déjà inscrit dont l'état a changé : un statut devenu faux, un point d'attente déjà levé. Ce genre de correction demande un regard, pas un grep.
  • Il ne scanne que trois blocs (bibliotheque/, notes/, recherches/). Un chantier vivant ailleurs, ou hors de votre ordinateur, lui échappe.

Ces limites ne sont pas des défauts : elles justifient le second outil. Le hook rattrape ce qu'on oublie d'ajouter ; la skill rattrape ce qu'on oublie de corriger.

04La skill — la méthode à la demande

Une skill est un mode d'emploi que l'assistant charge au moment où il en a besoin, et pas avant.

Plutôt que d'écrire la méthode complète dans les instructions permanentes (où elle pèserait en continu), on la range dans une skill que l'assistant ouvre quand la situation se présente. C'est là que vit la vraie méthode : quoi scanner, quoi vérifier, comment noter.

Où elle vit

L'en-tête du fichier (entre les deux lignes ---) décrit à l'assistant quand l'utiliser. Le corps décrit comment.

Comment elle se déclenche

Trois portes d'entrée, toutes décrites dans le description de l'en-tête : vous le dites (« mets à jour le pilotage »), l'assistant le propose de lui-même en fin de séance, ou le rappel du hook l'y invite. Dans les trois cas, l'assistant ouvre la skill et suit sa méthode.

05L'articulation : le flux bout à bout

Les deux outils et la consigne s'emboîtent ainsi.

  1. En fin d'échange, le hook compare les dossiers réels au tableau. S'il manque de la matière, il affiche son rappel.
  2. L'assistant, alerté par le rappel (ou par sa propre consigne de fin de séance), ouvre la skill pilotage.
  3. La skill lui fait traiter les deux fronts : la matière nouvelle et l'état périmé que le hook ne voyait pas.
  4. donnees.js est mis à jour, puis une sauvegarde est proposée.

Dans les instructions permanentes de l'assistant (le CLAUDE.md), il ne reste alors qu'une ligne : « en fin de séance, mets à jour le tableau via la skill pilotage ; le hook pilotage-drift double ce réflexe ». Le quand reste sous les yeux de l'assistant en permanence ; le comment n'est chargé qu'au bon moment.

06À adapter chez vous

Ces deux fichiers sont taillés pour notre dépôt. Avant de les réutiliser :

  • Dans le hook, remplacez bibliotheque, notes, recherches par les noms de vos dossiers de travail.
  • Dans la skill, ajustez la liste des blocs à scanner et le vocabulaire de vos statuts.
  • Vérifiez que les statuts cités correspondent à ceux de votre donnees.js (voir le mode d'emploi de base).

L'essentiel

  • Prolongement technique du mode d'emploi, réservé à Claude Code. La consigne permanente suffit à démarrer ; ceci la fiabilise.
  • Trois rôles séparés : le hook déclenche (automatiquement, à chaque fin d'échange), la skill porte la méthode (chargée à la demande), donnees.js fait foi.
  • Le hook détecte la matière absente ; il est aveugle à l'état périmé. La skill couvre ce second angle mort. Complémentaires, pas redondants.
  • Résultat : une seule ligne dans les instructions permanentes, une mise à jour fiable, et deux outils réutilisables et adaptables.
